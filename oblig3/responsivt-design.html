Responsivt design

Responivt design går ut på at det man utvikler (f. eks. nettsider, applikasjoner) passer og fungerer like godt på mobil og nettbrett, som på en laptop/PC. Det er ikke kun forskjellige skjermstørrelser man må tenkte på, men også at siden skal fungere like godt for en som bruker treg mobildata som en som er tilkoblet fiber.

Media queries

Man kan si at media queries er det som danner grunnlaget for responsivt design. Enkelt fortalt kan man bruke forskjellig CSS, som man baserer på spørringer til nettleseren. Med media queries kan man altså selv sette regler basert på f.eks. en skjermstørrelse. Eksempel på bruk:

@media (max-width: 940px) {
	.articles article {
		max-width: 75%;
	}
}

Media typer

Man kan si at media typer er forløperen til media queries, i forhold til responsivt design. Før media queries kom på banen, var det vanlig å lage egne stilark for de forskjellige enhetene. Det har tidligere blitt postet en artikkel om mediatyper på denne siden.

Fleksible størrelser

For å designe en nettside som er fleksibel med tanke på størrelsen, har man flere forskjellige metoder for å oppnå akkurat dette. Både Flexbox og Grid gjør det å lage responsive nettsider i dag super enkelt! Det har blitt skrevet om både flexbox og grid her tidligere.

Du kan lese mer om både flexbox og grid her.

Responsivitet på denne siden

Det største problemet og utfordringen, når det kommer til løsningen på denne siden, var at man skulle bygge på den forrige obligen. Da ble ikke mulig å gjøre "mobile-first", som er den måten vi har blitt anbefalt og utvikle responsive nettsider. Mobile-first går ut på at man først designer nettsiden for mobil, for så å gradvis style for større og større enheter. På denne nettsiden måtte jeg gjøre det omvendt. Jeg hadde en ferdig løsning for desktop, gjorde skjermen mindre helt til designet "kræsjet" for så å style til noe mer passende.

For responsivitet på denne nettsiden har jeg i hovedsak brukt media queries, flexbox og grid.

Git og Github

Git er teknologi, mens Github er en tjeneste som bruker teknologien Git. Git bidrar til å holde styr på prosjekter. For det gjør at man får oversikt på hva som blir gjort i et prosjekt, og ikke minst hvem som gjør det. Enkelt sagt kan man si at Git er et system for distribuert versjonskontroll.

Fordeler

En av de største fordelene er at man, som sagt, får versjonskontroll. Dette gjør blant annet at man kan rulle tilbake til en tidligere versjon ved feil, og samtidig beholde den nåværende versjonen. Noe som er utrolig verdifullt når man sitter og utvikler. Det er også nyttige når man samarbeider med andre. En annen stor fordel er at man får sikkerhetskopier av prosjektene sine, noe som er veldig viktig om man får slettet filer ved uhell, gjort noe feil eller blir utsatt for uforutsette hendelser som for eksempel tyveri av datamaskinen din. Med Git blir det også enklere å samarbeide med andre utviklere på et prosjekt, da man har mulighet til å "merge" kode sammen.

Mine tanker

Jeg har selv brukt Git og GitKraken de siste årene. I hovedsak med tanke på backup, men også fordi det gjør det så mye lettere å samarbeide med prosjekter, eller når jeg jobber på forskjellige enheter. Jeg syns at Git og tilhørende tjenester er noe alle burde bruke i webutviklingsverdenen. Det burde egentlig vært en selvfølge.
